# WeatherLink (Android)

Простое Android-приложение на Kotlin + Jetpack Compose.

Пользователь:
1. Вводит город.
2. Выбирает период прогноза (в днях).
3. Выбирает источник (Gismeteo / Яндекс Погода).
4. Нажимает кнопку и получает переход в браузер с подготовленным запросом.

## Запуск

1. Откройте проект в Android Studio (Hedgehog+).
2. Дождитесь синхронизации Gradle.
3. Запустите `app` на эмуляторе или устройстве.

## Примечание

Для универсальности используется поисковый URL на выбранный погодный ресурс с параметрами города и периода.

## PR и бинарные файлы

Если в системе code review появляется ошибка «Бинарные файлы не поддерживаются»,
исключите из PR файл `gradle/wrapper/gradle-wrapper.jar` (в этом репозитории он игнорируется).
При необходимости wrapper можно пересоздать локально командой:

```bash
gradle wrapper --gradle-version 8.7
```


## Частая ошибка при сборке

Если видите ошибку:

`resource style/Theme.Material3.DayNight.NoActionBar not found` или `resource style/Theme.Material3.DayNight not found`

это означает, что в теме использован стиль, которого нет в подключенных зависимостях Material3.
В проекте используется родитель темы `Theme.AppCompat.DayNight.NoActionBar` и подключена зависимость `androidx.appcompat:appcompat`, чтобы AAPT корректно находил XML-стиль темы.
После обновления файлов сделайте:

1. **File → Sync Project with Gradle Files**
2. **Build → Clean Project**
3. **Build → Rebuild Project**

После успешной сборки приложение появится на выбранном эмуляторе/устройстве, а не на рабочем столе ПК.
